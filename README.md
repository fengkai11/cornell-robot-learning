# Cornell Robot Learning 学习与实践

本仓库以 Cornell University **CS 4756/5756 Robot Learning (Spring 2026)** 为知识主线，记录课程学习、作业分析、ManiSkill 实验、论文阅读和阶段复盘。

- 课程主页：<https://www.cs.cornell.edu/courses/cs4756/2026sp/>
- 课程教师：Kuan Fang
- 实践平台：ManiSkill（必要时补充 PyTorch、Gymnasium、MuJoCo 等）

> 学习进度不以“看过多少课件”为标准，而以能否解释、形式化、推导、实现和验证为标准。

## 项目目标

完成本项目后，应至少形成以下能力：

1. 将机器人任务形式化为状态、观测、动作、策略、动力学、奖励和约束；
2. 判断传统控制、运动规划、模仿学习、强化学习、MPC 和混合方法的适用边界；
3. 阅读 Robot Learning 论文，识别问题定义、方法假设、实验设计与结论边界；
4. 在 ManiSkill 中复现核心方法，并设计可靠的对照实验；
5. 形成 BC、RL、MPC 与混合方法的统一评测框架；
6. 分别评价方法的工业落地价值与向家庭机器人迁移的价值。

## 学习原则

- 先建立整体框架，再进入公式和实现；
- 不把图像直接等同于状态，明确区分 state 与 observation；
- 不默认端到端学习优于控制和规划；
- 每种学习方法至少设置一个传统方法或简单学习方法基线；
- 不把一次成功运行当作算法有效；
- 不使用单一成功率支撑复杂结论；
- 区分感知误差、规划误差、控制误差、策略误差和评价误差；
- 对观点检查假设、遗漏条件和反例，区分事实、推断与待验证假设；
- 课程作业以独立思考为主，AI 仅用于讲解、检查、调试与复盘，并记录使用方式。

## 官方课程主线

| 阶段 | 核心主题 | 需要建立的能力 |
|---|---|---|
| Fundamentals | Robot Learning、机器人控制、MDP、深度学习 | 任务形式化与基础控制判断 |

## 每讲工作流

### 课前

1. 明确本讲试图解决的问题；
2. 补齐必要前置知识；
3. 提出 3–5 个带着问题阅读的问题；
4. 标出重点页与可暂时略读内容。

### 课中

采用统一顺序：

> 问题背景 → 直觉 → 严格定义 → 必要推导 → 算法流程 → 机器人实例 → 实现细节 → 失败模式

### 课后

每讲形成：

- 一句话概括；
- 核心概念与关键公式；
- 与相邻方法的区别；
- 隐含假设和失败模式；
- 工业价值与家庭迁移价值；
- ManiSkill 实验建议；
- 未解决问题与掌握度检查。

## 实验纪律

任何重要实验必须先填写 [`03-maniskill-labs/experiment-template.md`](03-maniskill-labs/experiment-template.md)，至少明确：

- 研究问题与待验证假设；
- 环境、任务和随机化设置；
- 状态、观测、动作与控制频率；
- 专家数据或交互数据来源；
- 传统基线、简单学习基线和目标方法；
- 控制变量、评价指标、随机种子；
- 分布内与分布外测试；
- 混杂因素和失败诊断顺序；
- 结论允许覆盖的范围。

## 掌握度标准

| 等级 | 判断标准 |
|---|---|
| L1 识别 | 知道术语的大致含义 |
| L2 解释 | 能用自己的话准确解释 |
| L3 形式化 | 能写出变量、假设、目标函数和关键公式 |
| L4 实现 | 能独立实现或复现简化版本 |
| L5 验证 | 能设计对照实验、诊断失败并限制结论范围 |

核心模块至少达到 L3–L4，重点模块力争达到 L5。
