# L01 课前导读：Introduction to Robot Learning

## 1. 本讲在课程中的位置

本讲不急于教授某个具体算法，而是建立整门课程的坐标系：

> 机器人需要感知世界并在世界中连续行动。Robot Learning 研究的是，哪些感知、预测与决策能力应从数据和交互中学习，以及这些学习模块如何与控制、规划和机器人本体组成闭环系统。

Cornell CS 4756/5756 Spring 2026 将课程概括为两类基本问题：

1. **Perception**：机器人如何通过不同传感器理解世界；
2. **Decision Making**：机器人如何考虑动作的后果并选择行为。

后续的控制、MDP、模仿学习、强化学习、MPC、感知与策略模型，都是对这两个问题的逐层展开。

## 3. 学习目标

完成本讲后，应能达到：

- 用自己的话解释什么是 Robot Learning；
- 画出“世界—感知—状态估计—决策—控制—世界”的闭环；
- 区分状态、观测、动作、控制输入、策略和动力学；
- 解释机器人学习为何比静态监督学习更难；
- 对一个机器人任务判断哪些部分适合传统方法，哪些部分可能需要学习；
- 正确理解 *The Bitter Lesson*，而不是把它简化为“只要堆算力即可”。

## 4. 必要前置知识

本讲不要求提前掌握完整的控制或强化学习，但应认识以下符号：

- 状态：$x_t$ 或 $s_t$；
- 观测：$o_t$；
- 动作：$a_t$；
- 策略：$\pi(a_t\mid o_t)$ 或 $\pi(a_t\mid s_t)$；
- 动力学：$p(s_{t+1}\mid s_t,a_t)$；
- 目标或代价：奖励 $r_t$、累计回报或控制代价。

暂时不要求推导 Bellman 方程、策略梯度或 MPC，这些将在后续讲次正式学习。

## 5. 带着问题阅读

阅读课件和 *The Bitter Lesson* 时，重点回答：

1. Robot Learning 与普通机器学习的根本区别是否只是“数据来自机器人”？
2. 为什么一个单步预测准确的模型，放进闭环后仍可能失败？
3. 机器人系统中，什么信息是可直接测量的，什么只能通过观测估计？
4. *The Bitter Lesson* 为什么推崇可以利用计算扩展的搜索和学习方法？
5. 在数据昂贵、安全约束强的机器人领域，通用学习方法与工程先验应如何配合？

## 6. 推荐阅读顺序

### 第一步：先看课程总览

确认课程要建立的不是某个单独算法，而是以下完整能力：

```text
感知问题的概率形式化
        +
机器人决策问题的 MDP 形式化
        +
IL / RL / MPC 的实现与比较
        +
分布偏移的识别与处理
        +
开放数据集和仿真平台上的评测
```

### 第二步：浏览 Lecture 01 课件

第一遍只寻找四类内容：

- 机器人需要完成哪些能力；
- 为什么现实机器人问题困难；
- 课程将覆盖哪些方法；
- 课件使用了哪些机器人案例来说明这些困难。

不要在第一次阅读时陷入具体算法名称。

### 第三步：阅读 *The Bitter Lesson*

阅读时将观点拆成三层：

1. **历史观察**：手工知识在短期有效，但常随规模增长而遇到平台期；
2. **方法偏好**：优先研究能持续吸收更多计算和数据的方法；
3. **非结论**：它并没有证明领域知识、物理结构、安全约束或系统分解永远无用。

### 第四步：回到机器人任务

选一个你熟悉的机械臂任务，回答：

- 哪些量天然可测？
- 哪些量需要感知估计？
- 哪些部分有可靠模型？
- 哪些误差会通过闭环累积？
- 学习方法的收益究竟来自哪里？
