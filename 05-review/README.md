# 阶段复盘

本目录用于检查知识是否真正掌握，并控制课程主线不被旁支问题打散。

## 建议复盘节点

- 完成 Fundamentals 后；
- 完成 Model-Free Decision Making 后；
- 完成 Model-Based Decision Making 后；
- 完成 Perception 后；
- 完成 Frontiers 后；
- 完成一组 ManiSkill 统一评测后。

## 每次复盘内容

### 1. 知识关系图

把本阶段概念连接到前置和后续主题，而不是孤立罗列。

### 2. 掌握度矩阵

| 主题 | 能解释 | 能形式化 | 能推导 | 能实现 | 能验证 | 证据 |
|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |

### 3. 概念漏洞

重点检查：

- 是否混淆 state、observation 和 belief；
- 是否混淆 planner、controller 和 policy；
- 是否混淆环境动力学与低层控制器闭环；
- 是否混淆训练损失、回报和评价指标；
- 是否混淆感知误差、控制误差和策略误差；
- 是否能说明算法成立的条件。

### 4. 综合题

综合题应要求把多个模块用于同一个机器人任务，例如：

> 设计一个存在目标位姿误差和接触不确定性的抓取任务系统，说明哪些部分使用状态估计、规划、控制、BC、RL 或 MPC，并给出对照实验。

### 5. 实验主线检查

- 当前实验是否服务于课程问题；
- 是否有明确传统基线；
- 是否存在不必要的复杂模型；
- 是否有 IID/OOD 和失败分类；
- 是否可以复现实验；
- 结论是否超出证据。

### 6. 下一阶段决策

- 必须补课的内容；
- 可以进入实现的内容；
- 可以进入论文阅读的内容；
- 暂不展开的旁支。

## 旁支管理

暂不属于当前课程主线的问题记录到 [`question-parking-lot.md`](question-parking-lot.md)，并标注与课程的关联点和重新进入条件。
