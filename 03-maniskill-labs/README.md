# ManiSkill 配套实验

本目录用于把课程概念转化为可以验证的机器人实验。

## 实验目标

实验不是为了证明某个新算法“更先进”，而是回答一个明确问题。例如：

- 已知目标位姿时，学习策略是否优于传统控制？
- BC 在初始位姿分布外为什么失败？
- 加入本体状态是否能减少视觉策略的数据需求？
- MPC 在模型存在偏差时是否仍优于开环轨迹？
- Domain Randomization 提升的是哪一种分布外鲁棒性？

## 第一阶段实验主线

```text
任务形式化
  ↓
传统控制基线
  ↓
状态策略 BC
  ↓
闭环数据修复（DAgger）
  ↓
无模型 RL
  ↓
MPC / 学习动力学
  ↓
视觉观测与状态估计
  ↓
序列策略和复杂任务
```

## 统一评价要求

除任务特定指标外，尽量统一记录：

- 成功率及置信区间；
- 最终位置与姿态误差；
- 碰撞率、越界率和安全约束违反率；
- 完成时间与动作步数；
- 控制平滑性与动作变化；
- 推理延迟和计算资源；
- 训练数据量与环境交互量；
- 不同随机种子均值和方差；
- IID 与 OOD 分组结果；
- 失败类型分布，而非只有总失败数。

## 基线纪律

每个学习实验至少选择两类基线：

1. **工程基线**：IK、轨迹规划、PID、操作空间控制、视觉伺服或 MPC；
2. **简单学习基线**：线性模型、MLP、单步 BC、简单 value/policy 方法。

复杂模型只有在简单基线不足时才引入。

## 失败诊断顺序

1. 环境定义与成功条件是否正确；
2. 坐标系、单位、控制模式和频率是否一致；
3. 专家或奖励是否真的产生目标行为；
4. 数据覆盖和训练测试划分是否合理；
5. 网络、损失、优化器和归一化是否正确；
6. 算法假设是否与任务不匹配；
7. 最后才考虑增加模型复杂度和训练规模。

## 工业与家庭迁移评价

每份实验报告增加两个独立小节：

### 工业落地

- 是否能用治具、标定、控制或规划更便宜地解决；
- 学习方法带来的真实收益；
- 数据采集、算力、部署、维护和安全成本；
- 异常恢复与持续运行能力。

### 家庭迁移

- 可复用的感知、控制、数据和策略能力；
- 对固定坐标系、固定物体和固定工装的依赖；
- 面对新对象、新场景和长时序任务时新增的困难。
